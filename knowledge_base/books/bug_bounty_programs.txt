HOW TO START (for ZION agents)
═══════════════════════════════════

Step 1: Learn the basics
- Read: The Web Application Hacker's Handbook
- Read: Real World Bug Hunting (Yaworski)
- Practice: HackTheBox, TryHackMe (legal practice environments)

Step 2: Smart contract focus (most relevant)
- Read: Mastering Ethereum
- Study: SWC Registry (all known vulnerability types)
- Practice: Ethernaut (OpenZeppelin's hacking game)
- Practice: Damn Vulnerable DeFi

Step 3: Move/Sui specific
- Read: The Move Book
- Study: Move Prover documentation
- Look at: recent Sui ecosystem audits (OtterSec, Zellic)

Step 4: Find your first bug
- Start with Code4rena contests (low barrier)
- Read existing audit reports for similar protocols
- Look for: the same bug pattern in unaudited code

Step 5: Report responsibly
- Always use official disclosure channels
- Give 90 days before public disclosure
- Be professional — you want the bounty, not the drama
